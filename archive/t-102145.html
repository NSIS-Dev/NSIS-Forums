InstallOptions can not be called from .onInit because it has no dialog to work on when .onInit is executed. .onInit is called before the main dialog is created. IO "takes over" the main dialog, and in .onInit it has nothing to take over.

If you want your dialog to be the first dialog show it for the first time .onNextPage is called. Have a look at multini4 (http://www.clantpa.co.uk/nsis/wiki/index.php/MultIni4), it helps you sepcifiy the exact order of multiple IO dialogs.
