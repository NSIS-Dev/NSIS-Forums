I have a log file from one of our customer's installations that shows the installer took the onInstFailed exit. The manual says (http://nsis.sourceforge.net/Docs/Chapter4.html#4.7.2.1.3) that can happen when he presses Cancel, or the script calls Abort, or it cannot extract a file. Are there any other causes? My script is run silently, so I know the user did not press Cancel; I don't have Abort in the script, either.

The onInstFailed routine looks like this:function .onInstFailed
Push "ResultCode=-11$\r$\n"
Call ErrorLog
SetErrorLevel -11
functionEnd
The parent (calling script) reported that the return code was -11. That line is the only place SetErrorLevel is used, too.

If your installer is trying to overwrite a file such as a running program, setup will show you a MessageBox, if you click Abort, the function ".onInstFailed" will be also called.
