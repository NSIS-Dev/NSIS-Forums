24th February 2006, 14:15
Well, it seems that every database type accepts different parameters for SQLConfigDataSource. According to MSDN (http://msdn.microsoft.com/library/en-us/odbcsql/od_odbc_c_99yd.asp), MS SQL doesn't accept a password there and only in SQLDriverConnnect. The Paradox (http://msdn.microsoft.com/library/en-us/odbc/htm/odbcjetparadoxsqlconfigdatasource.asp) driver, however, does accept PWD=password. Maybe it'll work with SQL Server as well...
