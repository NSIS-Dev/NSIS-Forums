To get the current user, you can use the UserInfo plug-in (http://nsis.sourceforge.net/Examples/UserInfo/UserInfo.nsi). But that'd require the user's password as well and you can't get that in a silent installer. You'd probably want to run the service with the SYSTEM user anyway and not the current user.
