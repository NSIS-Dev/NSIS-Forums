To see if the user has administrative privileges:
You could use the UserInfo plugin as described here (http://nsis.sourceforge.net/IsUserAdmin), or the system plugin as described here (http://nsis.sourceforge.net/Check_if_the_current_user_is_an_Administrator).

Another option would be to look for certain user privileges that relate to the admin account as described here (http://nsis.sourceforge.net/Enumerate_User_Privileges) or even enumerate all the available users as described here (http://nsis.sourceforge.net/User_Management_using_API_calls#Enumerate_all_users).

Finally you could just check the groups that the current user belongs to as described here (http://nsis.sourceforge.net/User_Management_using_API_calls#Get_User.E2.80.99s_Group.28s.29)

As for the desktop icon which is left behind, are you running the uninstaller using an admin context or a simple user? If you are uninstalling as a user try setting the write permissions for that file when you install your application so that everyone can delete it.
