There's no need to parse the version number and search for dots. When converting from the values returned by GetDLLVersion to that string, you get those numbers as a byproduct. Read the code comments in the conversion script (http://nsis.sourceforge.net/GetDllVersion_Command_Explained) and you'll see that you can find what you need in $R2, $R3, $R4 and $R5.
