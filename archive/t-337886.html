If you insert the macro twice, the EndRename label is also created twice. This is not allowed in NSIS, labels must be unique.

Use this:
!macro DllRename DllName
!define _LPrefix ${__LINE__}
...
goto ${_LPrefix}EndRename
...
${_LPrefix}EndRename:

!undef _LPrefix
!macroend
