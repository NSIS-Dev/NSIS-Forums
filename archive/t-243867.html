Could be, in which case you could call a function in NSIS which puts itself into a loop while your installer.exe is running. You can use the FindProcDLL (http://nsis.sourceforge.net/FindProcDLL_plug-in) plugin for that.

Like so:
Push setup.exe
Call WaitForInstaller


Function WaitForInstaller
Exch $R1
Push $R0
Loop:
Sleep 1000
FindProcDll::FindProc $R1
StrCmp $R0 1 Loop
Pop $R0
Pop $R1
FunctionEnd


Would be an idea to make the installer file name unique, in case the user already has a setup.exe process.
