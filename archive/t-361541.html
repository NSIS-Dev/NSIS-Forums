in the last couple of days, i've been working on a autocompleter for Sublime Text (http://www.sublimetext.com/) that i find worth mentioning here. it currently supports all nsis commands (naturally), all macros from "useful headers", all syntax from bundled plugins, and some other bits. other than the completion, it also comes with some scaffolding snippets to easily create empty scripts.

here's a video (https://dl.dropboxusercontent.com/u/353649/NSIS-Sublime-Text.m4v) of showing it in action (who spots the syntax error?;)).

the project is hosted on github (https://github.com/idleberg/NSIS-Sublime-Text) and there's also a wiki page (http://nsis.sourceforge.net/Sublime_Text_Packages) for it.

this is a work-in-progress, as i intend to extend its functionality over the time. looking forward to your feedback!

ps: the package is not yet available on package control (http://wbond.net/sublime_packages/package_control), i'm still waiting for my pull request to be accepted

The basic completions (https://github.com/idleberg/NSIS-Sublime-Text) and the add-on completions (https://github.com/idleberg/NSIS-Sublime-Text-Addons) have finally been added to Package Control (http://wbond.net/sublime_packages/package_control). The former supports all new NSIS 3.0a0, the latter supports the syntax of 165 NSIS plugins.

I've also commited the new syntax to the code highlighter (https://github.com/SublimeText/NSIS) and created a basic command lookup (https://github.com/idleberg/NSIS-Sublime-Text-Menu) context menu. All in all that should make Sublime Text (http://www.sublimetext.com/) a powerful NSIS editor.
