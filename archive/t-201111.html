10th December 2004, 11:27
To test if the temporary directory is valid, NSIS tries to create a temporary file in it using GetTempFileName. Because of this problem (http://forums.winamp.com/showthread.php?s=&threadid=130247&highlight=GetTempFileName), it is called a hundred times, with different prefixes, to avoid timing issues with other installers.

But I don't think that's the cause of the problem. Is it possible your temporary directory got filled up? It doesn't quite fit with your overclocking, but it's the best I can think of right now.
