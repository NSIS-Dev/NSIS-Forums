I've patched inetc (http://nsis.sourceforge.net/Inetc_plug-in) for LAN cable disconnect situation - WinInet returns EOF this case :) But for static files only (content-length header comes from server). Inetc is more stable then InetLoad (may be it's time to mark InetLoad 'depricated'), so please use Inetc.inec::get /TIMEOUT 5000 /RESUME "" "http://mydomain/myfile" "$TEMP\myfile"
