You can call directly the GetVersionEx (http://msdn.microsoft.com/library/default.asp?url=/library/en-us/sysinfo/base/getversionex.asp) API function to get the actual version of windows (in case you do not want to use the plugin). On NT machines you get service pack info at the szCSDVersion
member of the OSVERSIONINFO (http://msdn.microsoft.com/library/default.asp?url=/library/en-us/sysinfo/base/osversioninfo_str.asp) structure (it returns "miscellaneous information" on 9x machines). If you use the OSVERSIONINFOEX (http://msdn.microsoft.com/library/default.asp?url=/library/en-us/sysinfo/base/osversioninfoex_str.asp) structure then look also for wServicePackMajor and wServicePackMinor
