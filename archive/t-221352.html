i've written a mass-compiler for my tool xpy (http://xpy.whyeye.org/). besides the multilingual build there are also monolingual builds of it (smaller filesize!). with a growing number of available translations (seperate language files) the compilation got a growing problem, so with xpyle (http://prdownloads.sourceforge.net/xpy/xpyle-0.2-binsrc.zip?download) (source included) i automated the process.

it reads the language out of the filename of a language-file and uses it as ${LANG}. the language is being specified by that ${LANG} definition and passed to the compiler by the /D switch. the name of the language is then also used in the name of the zip-file, which includes the compiled nsis binary and some more specified files.

you can easily modify it for your own needs, enjoy.
