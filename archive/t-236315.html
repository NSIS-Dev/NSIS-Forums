you want only one nsis file?
i'd prefer different header files for every language.

i'd do it like that:
in you main.nsi:

!include ${lang}.nsh

(directly after you included the page macros, where you would normally place the !insertmacro MUI_LANGUAGE)

then in each language's header file:
!insertmacro MUI_LANGUAGE "<language>"
<add your custom language strings here>

now, you may compile the versions by using commandlineswitch like this:
/Dlang=english
/Dlang=french
/Dlang=german
