The only way you can tell is by checking the dates yourself using GetFileTime and GetFileTimeLocal. However, if it's a DLL that requires a reboot after replacement, you should compare it by version and not by date. You should also use the Library macros (http://nsis.sourceforge.net/Docs/AppendixB.html#B) instead of doing this manually.
