I use the Path Manipulation functions (http://nsis.sourceforge.net/archive/nsisweb.php?page=216&instances=0,11,211) in my installer and I discovered that if you push a directory on the stack that has a trailing backslash, the directory won't be found with the code:
# don't add if the path doesn't exist
IfFileExists $0 "" AddToPath_doneDirectories in the PATH environmental variable work with or without a trailing slash, and I like to add it to be consistent with how I treat other paths. Adding a slash shows that the path points to a directory, not a file.

When you add a directory to PATH, do you include a trailing slash, or am I alone here?
