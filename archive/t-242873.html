What kind of a message box is it exactly? Is the application generating it, or is the loader? If you put the DLL in the applications directory and in the PATH, it's probably the application's fault as that should be enough for the loader (http://msdn.microsoft.com/library/en-us/dllproc/base/loadlibrary.asp?frame=true). If it's not your application, write another application that only sets the working directory to the application's directory and executes the application and point the Run entry to this wrapper application.

GAC is, as far as I know, only for .NET applications. It doesn't sound related to me.
