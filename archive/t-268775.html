No need to apologize, I'm just trying to figure out what's needed here. If I am going to implement it some day, I need to know.

Currently, as images are stretched and icons already containing larger versions of themselves which Windows would probably automatically use, the transition should be relatively smooth. To get a perfect transition, you'd probably want to check the DPI and use a set of larger images so they won't appear stretched.

To identify selected DPI, GetDeviceCaps (http://blogs.msdn.com/oldnewthing/archive/2004/07/14/182971.aspx) can be used. MUI_WELCOMEFINISHPAGE_INIT will be where this will take affect and extract a different wizard image. MUI_HEADERIMAGE_INIT will also have to change for the header image. Probably a define named something like MUI_HEADERIMAGE_BITMAP_HIGH_DPI will need to be created which will contain the path to the higher DPI image. If empty, it should use the default MUI image for higher DPI.
