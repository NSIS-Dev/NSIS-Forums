The documentation for InstType claims that "Each type is numbered, starting at 1."

I think that the above-quoted line needs to be removed from the description of InstType, and the respective notes added to SectionIn and SetCurInstType; actually, that's where I thought to look first before I found the half-erroneous information in the InstType documentation. It would've saved me a lot of confusion regarding both commands if each one had a description of the type of index it wanted...
