I found a useful blog from Microsoft (what would we do without Google?)

Here's the link:
http://blogs.msdn.com/oldnewthing/ar...26/422076.aspx

Basically, it says that there is probably a program that has a bad icon resource reference. All entries in add/remove should be under HKLM\Software\Microsoft\Windows\CurrentVersion\Uninstall

And, you may also want to check the same path, but in HKCU.

(Just search for the string value named "DisplayIcon". Press 'F3' to repeat the search)

I'm just not sure exactly what would be considered "bad". The article says an icon index of "-1". I assume that just a bad icon library might also do it.
