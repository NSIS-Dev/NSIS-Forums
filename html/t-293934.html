Originally posted by Comperio
Plus, if the actual function relies on .NET, then you'd also need to make sure .NET framework is installed on the target PC.
I think every EXE or DLL created with a .NET language, such as C#, requires the .NET Framework. That's because such EXE/DLL files don't contain machine code, but some kind of bytecode. That code can't run on the CPU directly, it requires an interpreter. So without the .NET Framework it can't execute. That's similar to Java applications that require the JVM to run.

So if you want to call a .NET file, EXE or DLL, you must always make sure that the .NET Framework is installed on the system...
