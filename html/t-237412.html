Take a look at HKLM\Software\Microsoft\Windows\CurrentVersion\App Paths\Starlight.exe. It probably contains that path. When you use InstallDirRegKey and the registry key exists, that's what the directory page is going to show.

Because that is the path stored in the registry?
A good way to clean up the $INSTDIR path is to move it on and back off the stack (whenever the contents of the $INSTDIR variable is changed it is validated by the installer).


 Push $INSTDIR
 Pop $INSTDIR


I'm not quite sure why it isn't validated already, because if I use StrCpy $INSTDIR "C:\Temp\" in .onInit, C:\Temp\ is the path I get on the Directory page. However if I use the above code just afterwards it becomes C:\Temp instead (no trailing backstroke).

You need to put this in your directory pages' PRE function (use !define MUI_PAGE_CUSTOMFUNCTION_PRE DirectoryPre).
